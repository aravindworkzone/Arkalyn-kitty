# Project: Expense Tracker

## What this project does
A MERN-stack group expense-sharing app built on a pooled wallet model. Group members contribute money to a shared balance, and expenses are debited from that pool — eliminating circular debt tracking. Used by small groups (friends, roommates, teams) to manage shared spending transparently, with per-group roles, categories, and an immutable audit trail.

## Tech Stack
- **Frontend:** React 19 + Vite + Tailwind CSS + TypeScript
- **Backend:** Node.js + Express 5 + TypeScript
- **DB:** MongoDB (Atlas) + Mongoose
- **Auth:** JWT in HTTP-only cookies (single token, no refresh rotation)
- **State:** Redux Toolkit + RTK Query (caching, fetching, tag invalidation)

## Architecture / Request Flow

```
client → Express Router → Middleware (verifyToken → loadGroup → authorizeRole)
       → Controller (parse req, format res)
       → Service (all business logic + constraint validation)
       → Model (Mongoose schema + DB queries)
       → MongoDB (sessions for multi-doc writes)
```

Service layer is mandatory — controllers must not query models directly.

**Frontend flow:**
```
React page → RTK Query hook → base query (sends cookies) → Backend API
          ← cache invalidation via tag types (Auth, Group, Expense, Category, User)
          ← auto-redirect to /login on 401
```

## Folder Structure

```
Frontend/src/
  redux/api/       # RTK Query endpoint files per feature
  redux/store.ts   # Single store — RTK Query is the only reducer
  page/            # Route-level page components
  components/      # Reusable UI components
  interface/       # TypeScript interfaces
  utils/           # Helpers
  App.tsx          # All route definitions + ProtectedRouter

Backend/
  Router/          # Express route definitions per feature
  Middleware/      # auth.middleware.ts (verifyToken, loadGroup, authorizeRole)
  Controller/      # Request parsing and response formatting only
  Service/         # Business logic, validations, balance checks
  Model/           # Mongoose schemas
  Helper/          # Money.ts (cents conversion), AppError.ts
  DB/              # MongoDB connection
  Types/           # TypeScript type extensions (e.g., req.group, req.user)
```

## Key Conventions
- Always use `async/await`, never `.then()`
- All responses follow `{ success, message, data }` shape — use `AppError` for thrown errors
- Money is stored as **integer cents** in DB — use `toDBAmount()` before saving, `fromDBAmount()` before responding; never arithmetic on raw decimal values
- Expenses are **immutable** — no PATCH endpoint; delete and recreate to correct
- Every balance mutation must write to both `GroupTransaction` (audit) and `GroupEvent` (admin log)
- Soft deletes only — `GroupMember` and `Category` use `isDeleted: true`; always filter these out in queries
- Group `displayId` (e.g., `Grp-25-001`) is generated by the `Counter` sequence model — never set it manually

## Environment Variables

**Backend `.env`:**
- `PORT`, `MONGODB_URI`, `JWT_SECRET`, `FRONTEND_URL`

**Frontend `.env`:**
- `VITE_API_URL=http://localhost:5000/api`

Never hardcode. Always `process.env` (backend) or `import.meta.env` (frontend).

## What NOT to do
- Don't bypass the Service layer — controllers must not call Mongoose models directly
- Don't change the structure.
- Don't use `var` — only `const`/`let`
- Don't do multi-document writes without a Mongoose session (transaction)
- Don't add a PATCH endpoint for expenses — the design is intentionally immutable
- Don't query `GroupMember` or `Category` without filtering `isDeleted: false`
- Don't store money as floats — always convert to cents before saving
